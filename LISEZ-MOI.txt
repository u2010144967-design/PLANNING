1) AVANT DE METTRE EN LIGNE : LA SYNCHRONISATION (facultatif)
-------------------------------------------------------------
Sans rien faire, l'application fonctionne : les données restent dans
l'ordinateur qui l'utilise, et se transmettent par le bouton
« Enregistrer un fichier » / « Ouvrir un fichier ».

Pour que deux postes voient la même répartition en direct :

a) Dans Supabase, éditeur SQL, exécuter une fois :

   create table if not exists repartition_aout (
     id text primary key,
     data jsonb not null,
     auteur text,
     updated_at timestamptz not null default now()
   );
   alter table repartition_aout enable row level security;
   create policy "acces_public" on repartition_aout
     for all using (true) with check (true);

b) Ouvrir index.html avec le Bloc-notes, tout en haut du script,
   remplir les deux lignes :

   url:   "https://xxxxxxxx.supabase.co",
   cle:   "eyJhbGciOi…",              <- clé publique "anon"

   (Ces valeurs peuvent aussi être saisies dans l'application, via la
   pastille en haut à droite ; mais dans ce cas chaque poste doit les
   saisir de son côté.)


2) METTRE EN LIGNE SUR GITHUB
------------------------------
Déposer le dossier complet dans le dépôt u2010144967-design/PLANNING,
dans un sous-dossier REPARTITION_AOUT.

Adresse obtenue :
https://u2010144967-design.github.io/PLANNING/REPARTITION_AOUT/

C'est cette adresse qu'il faut donner au collègue : elle permet
l'installation en application et le fonctionnement hors connexion.


3) INSTALLER L'APPLICATION SUR UN ORDINATEUR
---------------------------------------------
Ouvrir l'adresse ci-dessus dans Chrome ou Edge, puis cliquer sur
« Installer l'application » en haut de la page (ou l'icône
d'installation dans la barre d'adresse).

L'application s'ouvre alors dans sa propre fenêtre, avec un raccourci
sur le bureau et dans le menu Démarrer, et s'ouvre même sans internet.

Sur téléphone : menu du navigateur -> Ajouter à l'écran d'accueil.


4) AVANT DE PARTIR EN CONGÉS
-----------------------------
- Terminer la répartition des 5 semaines (onglet « Préparer la
  répartition »). Le récapitulatif du mois doit afficher 0 dans la
  ligne « Sans chauffeur ».
- Écrire les consignes de chaque semaine : elles s'impriment sur les
  fiches des chauffeurs.
- Poser un verrou (bouton « Verrouiller ») : le collègue garde l'accès
  complet à la distribution des bons, mais la répartition ne peut plus
  être modifiée par erreur. Le code peut lui être communiqué en cas de
  besoin réel.
- Faire « Enregistrer un fichier » et garder le .json de côté : c'est
  la sauvegarde de secours.
- Imprimer « tout le mois » : 20 fiches A4 (4 chauffeurs x 5 semaines)
  à laisser dans le classeur, au cas où.
- Installer l'application sur le poste du collègue et lui montrer
  l'onglet « Distribuer les bons » + le bouton « Mode d'emploi ».
